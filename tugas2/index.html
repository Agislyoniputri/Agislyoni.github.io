01. Pengertian OOP

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berorientasi pada objek. Pendekatan ini membuat program lebih terstruktur, modular, serta mudah dikembangkan karena setiap bagian program dibangun dalam bentuk objek dunia nyata.

02. Konsep Dasar

  • Class: Blueprint atau cetakan untuk membuat objek.
  • Object: Instance nyata yang tercipta dari sebuah class.
  • Attribute: Representasi data melalui variabel di dalam class.
  • Method: Logika fungsional atau fungsi di dalam class.

03. Class Sederhana (Car)

Class adalah blueprint untuk membuat objek. Pada contoh ini, class Car memiliki atribut model dan color serta method start() dan stop().

DART SOURCE CODE
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Car {
  String? model;
  String? color;
  void start() {
    print('Car started!');
  }
  void stop() {
    print('Car stopped!');
  }
}
void main() {
  Car myCar = Car();
  
  myCar.model = "Sedan";
  myCar.color = "Merah";
  myCar.start();
  myCar.stop();
}

Proses menciptakan objek konkret dari class Car yang telah didefinisikan sebelumnya menggunakan fungsi main().

05. Constructor

Constructor memudahkan kita mengisi nilai atribut langsung saat objek diciptakan (inisialisasi).

DART SOURCE CODE
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Car {
  String model; 
  String color;

  Car(this.model, this.color);

  void start() {
    print('Engine Online: $model'); 
  }
}

void main() {
  Car myCar = Car("Sedan", "Merah");

  myCar.start(); 
  print(myCar.color); 
}

07. Encapsulation

Teknik keamanan data dengan menyembunyikan akses langsung ke variabel internal menggunakan prefix underscore.

DART SOURCE CODE
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Person {
  String _name; 
  int age;    

  Person(this._name, this.age);

  void setName(String name) => _name = name;
  
  String getName() => _name;
}

void main() {
  Person person = Person("Putri Lyoni Agis", 20);

  print(person.age); 
  person.setName("Agis Lyoni Putri");
  print(person.getName()); 
}

09. Product Class

Penggunaan Named Parameters dan Required untuk menciptakan objek yang lebih deskriptif dan aman.

DART SOURCE CODE
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Produk {
  final String id, nama; 
  final double harga; 
  final int stok;   
  final String kategori;

  Produk({
    required this.id,
    required this.nama,
    required this.harga,
    this.stok = 0,
    this.kategori = 'Umum',
  });
}

void main() {
  Produk laptop = Produk(
    id: "P001",
    nama: "Laptop Gaming",
    harga: 15000000,
    stok: 15,
    kategori: "Elektronik",
  );

  Produk smartphone = Produk(
    id: "P002",
    nama: "Smartphone Flagship",
    harga: 12000000,
    stok: 25,
  );

  Produk kamera = Produk(
    id: "P003",
    nama: "Kamera Mirrorless",
    harga: 20000000,
  );

  print(laptop.nama); 
  print(smartphone.nama); 
  print(kamera.nama); 
}
