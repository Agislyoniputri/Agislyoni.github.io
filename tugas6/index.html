Pengantar Generic

Generic adalah cara bikin kode yang fleksibel supaya bisa dipakai untuk berbagai tipe data tanpa harus nulis ulang. Dengan generic, kita bisa buat class, method, atau fungsi yang bisa bekerja dengan berbagai tipe data, tapi tetap aman karena tipe data itu ditentukan saat kita pakai.

Masalah Tanpa Generic

Tanpa generic, kita harus buat class atau fungsi baru untuk setiap tipe data yang mau kita pakai. Ini bikin kode jadi banyak dan susah di-maintain karena banyak duplikasi.

Kode ini berhasil karena nilai yang dimasukkan (42) sesuai dengan tipe yang diambil (int), bukan karena penggunaan dynamic itu aman.

contoh succes
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Box {
  dynamic data;
  Box(this.data);
}

void main() {
  var box = Box(42);
  int value = box.data; // Bisa error kalau bukan int
  print(value);
}
▶ klik output succes

Error terjadi karena tipe data tidak sesuai Di kode itu, data bertipe dynamic, artinya bisa menyimpan nilai apa saja tanpa dicek sejak awal. Saat kamu isi dengan "Halo", sebenarnya itu adalah String.

contoh error

Solusi Generic

Generic digunakan untuk menentukan tipe data dari awal, supaya data lebih aman dan terkontrol

Generic Class & Function

Kita bisa bikin class dan fungsi generic untuk berbagai tipe data, jadi kode kita lebih fleksibel dan bisa dipakai ulang tanpa duplikasi.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class MyClass {
  T data;
  MyClass(this.data);

  T getData() {
    return data;
  }
}

T myFunc(T value) {
  return value;
}

void main() {
  var a = MyClass(10);
  var b = MyClass("test");

  print(a.getData());
  print(b.getData());

  print(myFunc(100));
  print(myFunc("hello"));
}
▶ run

contoh Kelas Box

Kelas Box adalah contoh sederhana dari generic class di Dart. Dengan menggunakannya , kita bisa membuat Box yang bisa menyimpan data dari berbagai tipe, seperti int, String, atau bahkan tipe data kustom lainnya. Ini membuat kode kita lebih fleksibel dan bisa digunakan kembali tanpa harus membuat class baru untuk setiap tipe data.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Box {
  T? _value;

  Box(this._value);

  T? get value => _value;

  void setValue(T value) {
    _value = value;
  }
}

void main() {
  var box = Box(10);
  print(box.value);

  box.setValue(20);
  print(box.value);
}
▶ run

Multiple Type Parameters

Kita juga bisa menggunakan lebih dari satu parameter tipe dalam generic class atau fungsi. Ini memungkinkan kita untuk membuat struktur data yang lebih kompleks dan fleksibel, seperti pasangan nilai (pair) atau map yang bisa menyimpan berbagai tipe data.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Pair {
  K key;
  V value;

  Pair(this.key, this.value);

  void printPair() {
    print("Key: $key, Value: $value");
  }
}

void main() {
  var p1 = Pair("age", 20);
  p1.printPair();

  var p2 = Pair(1, 2.5);
  p2.printPair();
}
▶ run

Generic dengan extend

Kita bisa menggunakan keyword extends untuk membatasi tipe data yang bisa digunakan dalam generic class atau fungsi. Ini memungkinkan kita untuk memastikan bahwa hanya tipe data tertentu yang dapat digunakan, sehingga meningkatkan keamanan tipe dan mencegah kesalahan saat runtime.

void main() {

Type Inference (Inferensi Tipe)

Dart memiliki fitur type inference yang memungkinkan compiler untuk secara otomatis menentukan tipe data dari variabel berdasarkan nilai yang diberikan. Dengan menggunakan generic, kita bisa memanfaatkan type inference untuk membuat kode lebih bersih dan mudah dibaca, tanpa harus selalu menyebutkan tipe data secara eksplisit.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

T myFunc(T value) => value;

void main() {
  var a = myFunc(10);
  var b = myFunc("hello");

  print(a.runtimeType); // int
  print(b.runtimeType); // String
}
▶ run

Pembatasan Tipe dan manfaatnya

  • Lebih aman → hanya tipe yang sesuai yang bisa dipakai (mengurangi error)
  • Lebih jelas → kita langsung tahu tipe data yang digunakan
  • Bisa pakai method tertentu → karena tipe sudah dibatasi
  • Mencegah kesalahan → tidak bisa pakai tipe yang salah
  • Lebih terstruktur → kode jadi rapi dan mudah dikembangkan

Pembatasan Tipe pada Interface dan Class

Kita bisa menggunakan generic untuk membatasi tipe data yang dapat digunakan dalam interface atau class. Dengan cara ini, kita bisa memastikan bahwa hanya tipe data tertentu yang dapat diimplementasikan atau diwariskan, sehingga meningkatkan keamanan tipe dan mencegah kesalahan saat runtime.

contoh

// Kelas  : SI2A

void printIterable(T data) {
  for (var item in data) {
    print(item);
  }

void main() {

  printIterable>({"a", "b"});

Pembatasan Tipe dengan Class Abstract

Pembatasan tipe dengan class abstract berarti kita membatasi generic agar hanya bisa menggunakan turunan dari class abstract tertentu. Jadi, tipe data yang dipakai harus merupakan subclass (anak) dari class abstract tersebut.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

abstract class Animal {
  void makeSound();
}

class Dog implements Animal {
  void makeSound() => print("Woof");
}

class Cat implements Animal {
  void makeSound() => print("Meow");
}

class Cage {
  T animal;
  Cage(this.animal);

  void sound() {
    animal.makeSound();
  }
}

void main() {
  Cage(Dog()).sound();
  Cage(Cat()).sound();
}

Praktik 1 – Generic Class & Fungsi

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Repository {
  final List _data = [];

  void save(T item) {
    _data.add(item);
  }

  T getById(int id) {
    return _data[id];
  }

  List getAll() {
    return _data;
  }
}

// Fungsi swap
void swap(List list, int i, int j) {
  var temp = list[i];
  list[i] = list[j];
  list[j] = temp;
}

// Class Triple
class Triple {
  A first;
  B second;
  C third;

  Triple(this.first, this.second, this.third);
}

void main() {
  // =====================
  // TEST Repository
  // =====================
  var repo = Repository();
  repo.save("Data 1");
  repo.save("Data 2");

  print("Semua data: ${repo.getAll()}");
  print("Data index 0: ${repo.getById(0)}");

  // =====================
  // TEST swap
  // =====================
  var list = [1, 2, 3];
  swap(list, 0, 2);
  print("Setelah swap: $list");

  // =====================
  // TEST Triple
  // =====================
  var triple = Triple("Umur", 20, 3.14);
  print("Triple: ${triple.first}, ${triple.second}, ${triple.third}");

Generic vs Dynamic vs Object

Generic memberikan keamanan tipe yang lebih baik dibandingkan dengan dynamic atau Object. Dengan generic, kita bisa memastikan bahwa hanya tipe data tertentu yang dapat digunakan, sehingga mencegah kesalahan saat runtime. Sementara dynamic dan Object memungkinkan fleksibilitas, mereka tidak memberikan jaminan tipe, yang bisa menyebabkan error jika digunakan secara tidak hati-hati.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Box {
  T data;
  Box(this.data);
}

void main() {
  Box a = Box(10);
  dynamic b = 10;
  Object c = 10;

  print(a.data);
  print(b);
  print(c);
}
▶ run

Studi Kasus – Cache Generic

Kode ini mendemonstrasikan implementasi cache sederhana menggunakan generic class Cache. Cache ini bisa menyimpan data dari berbagai tipe, seperti String, int, atau bahkan tipe data kustom lainnya. Dengan menggunakan generic, kita bisa membuat cache yang fleksibel dan dapat digunakan kembali untuk berbagai jenis data tanpa harus membuat class baru untuk setiap tipe data.

contoh
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Cache {
  final Map _data = {};

  void set(String key, T value) {
    _data[key] = value;
  }

  T? get(String key) {
    return _data[key];
  }
}

void main() {
  var cache = Cache();
  cache.set("name", "Lyoni");

  print(cache.get("name"));
}
