01. Konsep Dasar Extension Methods

Fitur Extension Methods (diperkenalkan sejak Dart 2.7) adalah cara yang elegan untuk memperluas fungsionalitas suatu tipe data (baik bawaan SDK maupun class buatan sendiri) tanpa perlu mengubah source code aslinya atau membuat sub-class baru.

  • Menambahkan fungsionalitas eksternal: Sangat efisien ketika bekerja dengan pustaka pihak ketiga.
  • Type-Safe: Didukung penuh oleh analisis statik compiler Dart.
  • Berorientasi Objek: Pemanggilan method dapat dilakukan langsung dari instance tipe tersebut.

02. Keterbatasan Tanpa Extension Methods

Dahulu, developer sering memakai helper class statik untuk memodifikasi string atau data lainnya. Cara ini membuat kode terasa kurang berorientasi objek.

class StringUtils {
  static String capitalize(String text) {
    if (text.isEmpty) return text;
    return text[0].toUpperCase() + text.substring(1);
  }

  static bool isEmail(String text) {
    return text.contains('@');
  }
}

07. Utilities Tipe Dasar (num)

Kita dapat memasang extension pada tipe numerik num agar otomatis tersedia baik untuk `int` maupun `double`.

extension NumberExtensions on num {
  // Format Rupiah
  String toRupiah() {
    return 'Rp${toStringAsFixed(0).replaceAllMapped(
      RegExp(r'(\d{1,3})(?=(\d{3})+(?!\d))'),
      (match) => '${match[1]}.',
    )}';
  }

  // Cek rentang
  bool isBetween(num lower, num upper) {
    return this >= lower && this <= upper;
  }

  // Operasi pangkat
  num power(int exp) {
    var res = this;
    for (var i = 1; i < exp; i++) {
      res *= this;
    }
    return res;
  }
}

08. Penerapan Generics pada Extension

Extension juga dapat dikombinasikan dengan tipe generic <T> untuk memodifikasi list bertipe apa saja.

FILE: libgeneric_list_extension.dart
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

extension ListExtensions<T> on List<T> {
  // Ambil elemen dengan aman
  T? getOrNull(int index) {
    if (index < 0 || index >= length) return null;
    return this[index];
  }

  // Bagi list menjadi beberapa bagian (chunk)
  List<List<T>> chunk(int size) {
    var result = <List<T>>[];
    for (var i = 0; i < length; i += size) {
      result.add(sublist(i, i + size > length ? length : i + size));
    }
    return result;
  }

  // Filter nilai unik
  List<T> distinct() {
    var unique = <T>[];
    for (var element in this) {
      if (!unique.contains(element)) {
        unique.add(element);
      }
    }
    return unique;
  }
}

11. Extension untuk Custom Classes

Kita dapat menerapkan extension methods pada class buatan kita sendiri untuk memisahkan logika model dari utility tambahannya.

FILE: libproduct_extension.dart
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Product {
  final String name;
  final double price;

  Product(this.name, this.price);
}

extension DoubleRupiahFormatter on double {
  String toRupiah() {
    final intVal = toInt();
    String numStr = intVal.toString();
    String formatted = '';
    int cnt = 0;
    for (int i = numStr.length - 1; i >= 0; i--) {
      formatted = numStr[i] + formatted;
      cnt++;
      if (cnt % 3 == 0 && i != 0) {
        formatted = '.' + formatted;
      }
    }
    return 'Rp$formatted';
  }
}

extension ProductExtensions on Product {
  Product applyDiscount(double percent) {
    var newPrice = price * (1 - percent / 100);
    return Product(name, newPrice);
  }

  String get details => '$name - ${price.toRupiah()}';

  bool get isPremium => price > 500000;
}

extension StringValidators on String {
  bool get isValidEmail {
    return RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$').hasMatch(this);
  }

  bool get isValidPhone {
    return RegExp(r'^[0-9]{10,13}$').hasMatch(this);
  }

  bool get isValidSecurePassword {
    return length >= 8 &&
        contains(RegExp(r'[A-Z]')) &&
        contains(RegExp(r'[a-z]')) &&
        contains(RegExp(r'[0-9]'));
  }
}

extension IntValidators on int {

  bool get isEven => this % 2 == 0;
}

13. Ringkasan & Best Practices

Panduan penting ketika merancang Extension Methods:

  • Penamaan Extension: Disarankan menggunakan format PascalCase dan deskriptif (misal: StringValidators).
  • Prinsip Modularitas: Letakkan extension sejenis di file terpisah agar pengelolaan file lebih mudah.
  • Hindari Overuse: Jangan mengubah semua kelas pembantu statik menjadi extension method jika kurang dibutuhkan.

🌸 Bonus Soal 1: Konflik Nama Antar Extension

Studi Kasus: Dua buah extension menargetkan tipe String yang sama dengan nama method yang sama yaitu toInt() namun beda tipe kembalian.

FILE: test_bonus_1.dart 
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

extension StringNumber on String {
  int toInt() {
    return int.parse(this);
  }
}

extension StringDouble on String {
  double toInt() {  // ← Nama method sama, return type berbeda
    return double.parse(this);
  }
}

void main() {
  String angka = '123';
  
  // print(angka.toInt()); // ❌ ERROR: Ambiguous member lookup

  // Solusi: Pemanggilan Eksplisit
  int hasilInt = StringNumber(angka).toInt();
  double hasilDouble = StringDouble(angka).toInt();

  print('Hasil StringNumber (int): $hasilInt');
  print('Hasil StringDouble (double): $hasilDouble');
}
Penjelasan Analitis:

1. Apa yang terjadi? Compiler akan menolak untuk mengompilasi kode tersebut (Compile-time error) karena pemanggilan angka.toInt() bersifat ambigu.

2. Penyebabnya? Dart tidak mendukung *method overloading* pada pencarian extension implisit. Karena kedua extension mengarah pada target bertipe data sama (String) dengan nama method yang sama (toInt()), compiler tidak tahu implementasi mana yang harus dipakai meskipun tipe kembaliannya berbeda.

3. Solusi: Lakukan pemanggilan secara eksplisit dengan menuliskan nama extension di depan variabel target, seperti StringNumber(angka).toInt() dan StringDouble(angka).toInt().

🌸 Bonus Soal 2: Member Private & Extension

Studi Kasus: Mencoba mengakses field private berawalan _ milik suatu class dari dalam tubuh extension.

FILE: test_bonus_2.dart 
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Counter {
  int _value = 0;  // Private pada level file/library

  void increment() {
    _value++;
  }

  // Solusi: Buat getter public agar bisa diakses di file terpisah
  int get value => _value;
}

extension CounterStats on Counter {
  int getValue() {
    // Berhasil dikompilasi jika class dan extension diletakkan di file yang sama
    return _value; 
  }
}

void main() {
  var c = Counter();
  c.increment();
  print('Value: ${c.getValue()}');
}
Penjelasan Analitis:

1. Mengapa gagal dikompilasi di file terpisah? Visibilitas private di Dart (menggunakan awalan _) didasarkan pada level library (file), bukan class. Jika file extension dideklarasikan terpisah dari file class Counter, maka variabel _value tidak dapat terlihat atau diakses oleh extension.

2. Solusi: Letakkan class Counter dan extension CounterStats di dalam berkas/file yang sama, atau buat sebuah getter public (contoh: int get value => _value;) di dalam class asli agar extension di file terpisah dapat membacanya lewat getter tersebut.

🌸 Bonus Soal 3: Prioritas Method Asli vs Extension

Studi Kasus: Class memiliki method bawaan asli dengan nama yang persis sama dengan method di dalam extension.

FILE: test_bonus_3.dart 
// Nama   : Agis Lyoni Putri
// Nim    : 251410022
// Kelas  : SI2A

class Greeting {
  String say() => 'Hello from class';
}

extension GreetingExt on Greeting {
  String say() => 'Hello from extension';
}

// Mendefinisikan extension khusus nullable tipe
extension GreetingNullableExt on Greeting? {
  String sayNullable() {
    if (this == null) return 'Hello from extension (Null)';
    return 'Hello from extension (Not Null)';
  }
}

void main() {
  Greeting? obj = Greeting();
  
  // 1. Memanggil method asli bawaan kelas (prioritas utama)
  print('Panggilan Implisit: ${obj?.say()}'); // Output: Hello from class
  
  // 2. Memaksa memanggil method extension (Panggilan Eksplisit)
  print('Panggilan Eksplisit Extension: ${GreetingExt(obj!).say()}'); 

  // 3. Menangani nilai null
  obj = null;
  print('Ketika Null: ${obj?.say()}'); // Output: null
  print('Nullable Extension: ${GreetingNullableExt(obj).sayNullable()}');
}
Penjelasan Analitis:

1. Output dari program asli? Output yang keluar adalah Hello from class.

2. Mengapa demikian? Aturan compiler Dart menetapkan bahwa instance method asli milik kelas selalu memiliki prioritas pemanggilan yang lebih tinggi dibandingkan extension method jika keduanya memiliki nama signature yang sama.

3. Jika obj bernilai null? Pemanggilan obj?.say() akan mengembalikan nilai null karena operator null-aware mencegah evaluasi method tanpa crash.

4. Cara memastikan extension method yang dijalankan? Panggil extension secara eksplisit dengan format pembungkus, misalnya: GreetingExt(obj!).say().
